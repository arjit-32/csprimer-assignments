# Git Assignments

## Assignment-1: Initialize a New Repository With a Clean Project Structure  
_🎯 Goal: Set up a new repository with a proper folder structure, `.gitignore`, and semantic commit messages._  

**🧠 Skills:** Repo setup, project hygiene.  

---

## Assignment-2: Create and Use Feature Branches  
_🎯 Goal: Create a feature branch, commit changes in logical steps, and merge back cleanly._  

**🧠 Skills:** Branching strategy, meaningful commit discipline.  

---

## Assignment-3: Practice Merge vs Rebase  
_🎯 Goal: Create diverging branches, resolve conflicts twice: once via merge and once via rebase._  

**🧠 Skills:** Rebasing safely, conflict resolution.  

---

## Assignment-4: Perform an Interactive Rebase  
_🎯 Goal: Squash multiple commits into a single clean commit._  

**🧠 Skills:** Commit rewrite, history cleanup.  

---

## Assignment-5: Recover Code Using git reflog or git restore  
_🎯 Goal: Intentionally delete a branch or commit, then restore it._  

**🧠 Skills:** Preventing disasters, confidence in experimentation.  

---

## Assignment-6: Create and Maintain a Fork  
_🎯 Goal: Fork a repository, sync upstream changes, and create pull requests._  

**🧠 Skills:** Open-source workflow.  

---

## Assignment-7: Write a Pull Request With Reviewer Context  
_🎯 Goal: Submit a pull request with a checklist, description, linked issue, and migration notes._  

**🧠 Skills:** Communication, code review etiquette.  

---

## Assignment-8: Resolve a Merge Conflict in a Real File  
_🎯 Goal: Simulate multiple contributors editing the same file and resolve conflicts._  

**🧠 Skills:** Real-world conflict resolution.  

---

## Assignment-9: Configure and Use Git Hooks  
_🎯 Goal: Create a pre-commit hook to run formatting or linting automatically._  

**🧠 Skills:** Automation, commit quality enforcement.  

---

## Assignment-10: Create Release Tags and Versioning  
_🎯 Goal: Create annotated tags for releases following semantic versioning (SemVer)._  

**🧠 Skills:** Release versioning and historical tracking.  

---

## Assignment-11: Use .gitignore and .gitattributes Correctly  
_🎯 Goal: Ignore compiled files, logs, and use `.gitattributes` for line ending normalization._  

**🧠 Skills:** Cross-platform consistency.  

---

## Assignment-12: Manage Binary Files and Large Assets  
_🎯 Goal: Use Git LFS to manage large files and test repository size differences._  

**🧠 Skills:** Optimizing version control for media/data.  

---

## Assignment-13: Analyze Commit History  
_🎯 Goal: Use `git log`, filters, blame, shortlog, and search filters to analyze commit history._  

**🧠 Skills:** Debugging and auditing.  

---

## Assignment-14: Create a Branching Model for a Team  
_🎯 Goal: Design a branching model like Git Flow, GitHub Flow, or trunk-based development._  

**🧠 Skills:** Workflow design and reasoning.  

---

## Assignment-15: Cherry-Pick Commits Across Branches  
_🎯 Goal: Take specific fixes from another branch without merging everything else._  

**🧠 Skills:** Hotfix workflows.  

---

## Assignment-16: Stash and Restore Work  
_🎯 Goal: Use stashing for temporary context switching, including named and partial stashes._  

**🧠 Skills:** Interrupt-driven developer reality.  

---

## Assignment-17: Practice Bisecting to Find a Bug  
_🎯 Goal: Introduce a bug intentionally and use `git bisect` to find the breaking commit._  

**🧠 Skills:** Debugging and traceability.  

---

## Assignment-18: Clone and Audit an Open-Source Repository  
_🎯 Goal: Study commit structure, tags, pull request hygiene, hooks, and versioning patterns._  

**🧠 Skills:** Real-world conventions and learning from mature codebases.  

---

## Assignment-19: Sync a Long-Lived Local Branch With Main  
_🎯 Goal: Resolve drift using rebase or merge strategy and document trade-offs._  

**🧠 Skills:** Long-term branch management.  

---

## Assignment-20: Configure CI/CD Repo Settings (If using GitHub/GitLab)  
_🎯 Goal: Enable branch protection rules, required reviews, signed commits, or status check gating._  

**🧠 Skills:** Production-grade repository governance.
